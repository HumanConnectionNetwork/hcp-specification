# Contributing to Humanitarian Connection Protocol (HCP)

First of all, thank you for your interest in contributing to the Humanitarian Connection Protocol.

HCP is an open standard designed to improve humanitarian coordination through interoperable information exchange.

Whether you are a software engineer, humanitarian worker, researcher, designer or volunteer, your contributions are welcome.

---

# Our Mission

We believe humanitarian information should be:

- Open
- Interoperable
- Verifiable
- Accessible
- Human-centered

Our goal is to create an open protocol that allows humanitarian organizations, governments, volunteers and digital platforms to work together using a common language.

---

# Ways to Contribute

There are many ways to help.

## Improve the Specification

Help improve:

- protocol definitions
- terminology
- examples
- diagrams
- documentation
- JSON Schemas

---

## Report Issues

If you find:

- inconsistencies
- ambiguous wording
- missing definitions
- technical problems

please open a GitHub Issue.

---

## Propose New Features

Major protocol changes should begin with a proposal.

Future versions of HCP will use the **HCPP (Humanitarian Connection Protocol Proposal)** process.

Until then, proposals can be discussed through GitHub Issues.

---

## Improve Examples

Real-world examples are extremely valuable.

You can contribute:

- sample records
- API examples
- implementation examples
- interoperability scenarios

---

## Improve Documentation

Clear documentation is one of the most valuable contributions.

Not every contribution requires writing code.

---

# Development Philosophy

When contributing to HCP, please keep these principles in mind:

- Simplicity over complexity.
- Backward compatibility whenever possible.
- Humanitarian impact comes first.
- Vendor neutrality.
- Open collaboration.
- Privacy by design.

---

# Pull Requests

Before submitting a Pull Request:

- Read the relevant specification.
- Search existing Issues.
- Keep changes focused.
- Explain the motivation behind the change.
- Include examples whenever appropriate.

Small Pull Requests are preferred over very large ones.

---

# Discussions

Technical discussions are encouraged.

Respectful disagreement is part of building a better protocol.

Our objective is not to "win" discussions.

Our objective is to build the best possible open standard.
