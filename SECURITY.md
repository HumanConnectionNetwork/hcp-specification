# Security Policy

## Security Philosophy

The Humanitarian Connection Protocol (HCP) is designed with security, privacy and trust as fundamental principles.

Humanitarian information may involve vulnerable individuals, emergency situations and sensitive operational data.

For this reason, security is considered a core part of the protocol, not an optional feature.

---

# Reporting Security Issues

If you discover a security vulnerability affecting:

- the protocol specification
- the reference implementation
- official SDKs
- official tools

please report it responsibly.

Do **not** open a public GitHub Issue for undisclosed security vulnerabilities.

Instead, contact the maintainers privately.

A dedicated security contact and disclosure process will be published as the project grows.

---

# Responsible Disclosure

We ask security researchers to:

- Allow maintainers reasonable time to investigate.
- Avoid public disclosure until a fix or mitigation exists.
- Provide enough technical information to reproduce the issue.

We are committed to acknowledging responsible disclosures.

---

# Security Goals

HCP is designed to support:

- Confidentiality where appropriate.
- Integrity of humanitarian information.
- Authenticity of records.
- Verifiable data provenance.
- Secure interoperability between systems.

---

# Privacy

The protocol should minimize unnecessary personal information.

Implementations are encouraged to:

- collect only required data
- anonymize information whenever possible
- encrypt sensitive communications
- comply with applicable privacy regulations

Privacy is a shared responsibility between the protocol and its implementations.

---

# Trust Model

HCP does not assume that every participant is trusted.

Instead, implementations should support mechanisms for:

- verification
- provenance
- reputation
- auditability

Future specifications will define these mechanisms in detail.

---

# Cryptographic Recommendations

Whenever cryptographic functionality is required, implementations should use modern, well-established algorithms and libraries.

Custom cryptographic algorithms should never be used.

---

# Future Security Specifications

Future HCP specifications are expected to cover topics such as:

- Digital Signatures
- Record Integrity
- Node Authentication
- Secure Synchronization
- Identity Verification
- Trust and Reputation Models

---

# Scope

This policy applies to:

- hcp-specification
- official reference implementations
- official SDKs
- official tools maintained by the Human Connection Network

Third-party implementations are responsible for their own security practices.

---

# Security Is a Shared Responsibility

The protocol provides the foundation.

Secure implementations require responsible development, careful deployment and ongoing community collaboration.

Together we can build trustworthy humanitarian infrastructure.
